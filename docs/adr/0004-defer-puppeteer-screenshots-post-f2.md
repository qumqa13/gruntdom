# ADR-0004 — Defer F1-T7 + F1-T8 (Puppeteer-driven snapshots) to post-F2

- **Status**: Accepted (2026-05-10 — Oskar signed off after reviewing §5 trigger criteria; F1 formally closed)
- **Date**: 2026-05-10
- **Decision drivers**: F1 close pressure, downstream-consumer absence, Puppeteer-infra cost-of-introduction, ADR-0002 §2.1 (T7 row), F1 plan §F1-T7 / §F1-T8.
- **Supersedes**: none. Refines the F1 task table in `docs/plans/3d-viewer-data-layer.md`.
- **Will be reviewed**: at F2 close (when the 3D viewer ships for Balice 773), and again at F3 kick-off (when `<ProvenanceBadge>` modal starts implementation).

---

## 1. Context

The F1 plan calls for two Puppeteer-headless screenshot workers:

- **F1-T7** `apps/workers/jobs/generate-provenance-screenshots.ts` — per-layer Geoportal screenshots (kataster, MPZP, ortofoto, NMT, klasy gleby, sieci uzbrojenia) cropped to plot bbox + 50% padding, output to R2 path `provenance/{slug}/{layer}-{fetchedAt}.png`. Consumed by **F3-T4** `<ProvenanceBadge>` clickable modals.
- **F1-T8** `apps/workers/jobs/generate-3d-preview-thumbnail.ts` — single ortofoto crop centered on plot centroid + SVG-overlaid polygon, output to R2 path `previews/{slug}-3d-preview.jpg`, cache 30 days. Consumed by an opt-in lazy-load placeholder for the **F2** Cesium viewer.

F1's other deliverables (T1-T6 + T9-T11) close the **type-system** and **data-layer** guarantees of the verified-data architecture. By 2026-05-10, T1-T6 are landed (rebased on Path B), T9 (ESLint provenance rule) and T10 (synthetic fixtures) closed via parallel agents, T11 (real-network worker smoke against Balice 773) is committed at `a3eb369` with golden ok-gate verified.

T7 and T8 are **presentation-layer artifacts** generated by a worker. They produce assets that are **only consumed downstream**:

| Worker | Consumer | Phase | Status of consumer today |
|---|---|---|---|
| F1-T7 (provenance screenshots) | F3-T4 `<ProvenanceBadge>` modal | F3 | not implemented; full F2 must ship first |
| F1-T8 (3D preview thumbnail) | F2 lazy-load placeholder | F2 | not implemented; placeholder may turn out unnecessary if Balice 773 viewer renders fast enough |

Both workers depend on **Puppeteer headless Chrome**, which carries its own cost of introduction:

- ≈ 600 MB Chromium download in build pipeline (or vendored binary in container).
- IP-based rate-limit considerations on Geoportal (the same constraint that drove ADR-0002 §2.8 token bucket for WMS).
- Worker error handling for browser timeout, page-load races, layer-load races, font / map-tile flakiness.
- Visual regression discipline (a screenshot is only useful as a frozen trust signal if it's reproducible; small font / tile-version drift produces visually-different PNGs that look identical to a reviewer).

Estimated true cost of T7+T8 if landed in F1: **1.5-2 engineer-days** (infra setup + happy-path implementation + 2-3 layer-specific edge cases + retry/error paths). Estimated downstream value if landed today: **zero** — neither consumer exists.

---

## 2. Decision

**Defer both F1-T7 and F1-T8 to post-F2 demo.**

Concretely:

1. F1 closes after T11 + ADR-0004 (this document) accepted. No T7, no T8 inside F1 deliverable scope.
2. The two task ID slots stay reserved (do not renumber other tasks). F1 task table in `docs/plans/3d-viewer-data-layer.md` is updated with a `[deferred — see ADR-0004]` annotation against T7 and T8 rows; the rows are not deleted, so cross-references from session handoff and prior commits remain valid.
3. The two tasks are **re-classified as F2-tail (T8) and F3-head (T7)** at the moment their consumers' tickets are written. Reclassification is editorial — a docs-only change in the plan, no commit history rewrite.

---

## 3. Why now (= why this is the right boundary)

**Cost of introduction is front-loaded; cost of late introduction is amortised.** Puppeteer setup pays back over 4-6 layers; today we'd amortise it over zero. F2 ships the 3D viewer for ONE plot and may discover that the screenshot infra needs to live on Cesium-side anyway (e.g. capture a pre-rendered Cesium frame instead of a Geoportal page) — which would re-tool T7 entirely. Settling Puppeteer infra now risks throwing it away in 2 weeks.

**No data-layer guarantee is weakened by deferral.** F1's job is to make plot data verifiable and validated. T7 makes the *evidence* clickable; T8 makes the 3D viewer *load faster*. Both are user-experience enhancements over a layer (data correctness) that's already complete after T1-T6 + T9-T11.

**F1 close lets F2 start.** F2-T1 is a 1-day timeboxed spike on `cesium-terrain-builder` Windows toolchain. Holding F1 open for 1.5-2 days of T7+T8 work that produces no demo-visible output postpones the F2 spike by the same amount. Earlier failure information from the spike (whether ctb works on Windows, whether Tier 2 Cesium Ion fallback is needed) is more valuable than earlier T7+T8 assets.

**No regression risk.** T7+T8 produce write-only side effects (screenshots in R2). The runtime app does not consume them today. There is no test-suite gap that T7+T8 would close (synthetic fixtures cover the data path; real-network smoke covers the worker path). Deferring is safe; landing it would not change a single test count.

## 4. Alternatives considered

### 4.1 Ship T7+T8 in F1 as planned

- **Pro**: F1 fully closed per original plan; no deferral.
- **Con**: 1.5-2 days of work with zero downstream consumer; risk of re-tooling once F2 reveals that Cesium-side capture is the right primitive; delays F2 spike with no demo-visible benefit.
- **Verdict**: rejected.

### 4.2 Ship T7 only (provenance screenshots), defer T8 (3D preview)

- **Pro**: T7 is more justifiable in isolation — provenance screenshots are part of the verified-data narrative, and could be useful even without the F3 modal (e.g. as static R2 artefacts a developer can manually link in QA).
- **Con**: Same Puppeteer infra cost as both. No saving on the costly part. Saves only the per-layer iteration time. Still pre-commits to the Geoportal-page-capture primitive when F2 may reveal Cesium-frame-capture is actually required.
- **Verdict**: rejected. If we pay for Puppeteer infra, we should ship both; if we're not certain about the primitive, we should ship neither.

### 4.3 Defer both (= decision in §2)

- **Pro**: Zero work in F1 scope, zero risk to F2 timeline, infra paid only once when there's a real consumer to amortise it.
- **Con**: F1 closes with two task slots formally deferred (visible "loose ends"); requires this ADR + a follow-up plan-doc edit.
- **Verdict**: **accepted**.

### 4.4 Cancel T7+T8 entirely (move to dropped scope)

- **Pro**: cleanest task list.
- **Con**: ProvenanceBadge modal needs frozen screenshots from somewhere; cancelling the worker doesn't cancel the requirement, just hides it. Better to defer with explicit revisit triggers (§5) than to silently drop.
- **Verdict**: rejected.

## 5. Trigger to revisit (= when this ADR converts from "Proposed/Accepted" to "Superseded")

Each task has its own revisit trigger. Both must fire independently — landing one does not unlock the other.

### 5.1 T7 (provenance screenshots) — revisit when **any** of:

- **Primary**: F3-T4 `<ProvenanceBadge>` ticket is opened (= a designer/PM signs off on the modal copy and interaction). At that point T7 becomes a hard dependency and must land before F3-T4 implementation.
- **Secondary**: F2 reveals that Cesium-frame capture rather than Geoportal-page capture is the right primitive. Re-scope T7 to use the Cesium primitive; this ADR is partially superseded.
- **Tertiary** (lowest priority): a Puppeteer worker lands for an unrelated reason (e.g. F3-T6 PDF dossier export). Bundle T7 with that workstream — share Chromium cache, share retry primitives, share visual-regression discipline.

### 5.2 T8 (3D preview thumbnail) — revisit when **any** of:

- **Primary**: F2 viewer ships and post-launch metrics show that the Cesium initial-load time on Balice 773 measurably hurts UX (LCP > 4 s on a P75 cold cache, or user-reported "blank screen" feedback). Then T8 lands as the lazy-load placeholder strategy.
- **Secondary**: F2.5 generalisation begins (3D viewer for plots beyond Balice 773). T8 may be needed as a per-plot preview in the catalog grid even before any single plot's detail page renders. At that point T8 is a precondition for F2.5, not for F2.
- **Tertiary**: F2 viewer ships and the showcase plot's first-paint is fast enough to skip a placeholder (single hero image fades into Cesium without an interim). T8 becomes unnecessary; this ADR converts to "Cancelled-T8" rather than "Superseded".

### 5.3 Hard escape hatch

If Oskar at any point says "ship T7+T8 next sprint", this ADR converts to Superseded. The deferral is a default to optimise F1-close velocity, not a binding constraint.

## 6. Consequences

### 6.1 Immediate

- F1 task table (`docs/plans/3d-viewer-data-layer.md` §F1-T7, §F1-T8 rows) gets a `**Status:** Deferred — see ADR-0004` annotation. Editorial PR, no logic change.
- F1 close PR description references this ADR under "Deferred work".
- F2-T1 spike kicks off without further prerequisites.

### 6.2 Near-term (F2 in flight)

- ADR-0002 §2.8 (Puppeteer screenshot worker config) remains the architectural reference for when T7 lands. No premature alternative architecture is documented.
- The `provenance` field on `PlotGeometry` (added by 053a662 / now `bf052f1` post-rebase) and `ProvenanceMap<Plot>` on `Plot` (same commit) **are not orphaned by this deferral** — they describe the data shape that the F1 worker emits today (sidecar JSON `validators[].gateLevel`, `geometry.terytId`, etc.). They will be **populated** with screenshot R2 keys when T7 lands; they are not blocked on T7.

### 6.3 Long-term (post F2 demo)

- This ADR's revisit triggers (§5) are checked at F2 close gate. Whichever fires first converts the ADR to a referenced decision.
- The reserved task slots T7 + T8 are renamed in the plan doc to F3-T4-deps and F2-T11 (or wherever they end up), but their numbers can stay if the cross-references are too cumbersome to update.
